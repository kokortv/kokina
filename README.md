# Kokina

PWA для учета цен в реальных магазинах: открываете визит в конкретный магазин, добавляете товары пачкой, приложение сохраняет записи локально, автоматически подбирает категорию и умеет синхронизировать данные в Google Sheets.

## Запуск

```bash
python3 -m http.server 4173
```

Откройте `http://localhost:4173`.

## Google Sheets напрямую

1. В Google Cloud Console создайте OAuth Client ID типа `Web application`.
2. В Authorized JavaScript origins добавьте `http://localhost:4173`.
3. Включите Google Sheets API для проекта.
4. В приложении откройте настройки, войдите через Google и нажмите `Создать таблицу`.

Приложение использует Google Identity Services token model и REST Google Sheets API. Записи хранятся локально до успешной синхронизации, поэтому ими можно пользоваться без связи.

При синхронизации для каждой конкретной торговой точки создается отдельный лист. Название листа строится из названия магазина и координат, например `Spar 41.725000,44.744000`. Колонки листа: штрихкод, название товара, дата внесения в базу, цена, количество, валюта, координаты магазина, название магазина, адрес/локация, страна, категория, единица, заметка, ID записи, дата синхронизации.

## Общий режим через Apps Script

1. Откройте Kokina Google Sheet.
2. Выберите `Extensions -> Apps Script`.
3. Скопируйте код из `apps-script/Code.gs`.
4. В начале файла проверьте список `ADMINS`.
5. Нажмите `Deploy -> New deployment -> Web app`.
6. Выберите `Execute as: Me`.
7. Для доступа выберите вариант, который позволяет Google-пользователям вызвать Web App.
8. Скопируйте Web App URL и вставьте его в настройках Kokina в поле `Apps Script Web App URL`.

В этом режиме таблица не требует edit-доступа для пользователей. Все Google-пользователи могут добавлять новые цены через приложение, а редактирование и удаление существующих строк разрешено только email из `ADMINS`.
